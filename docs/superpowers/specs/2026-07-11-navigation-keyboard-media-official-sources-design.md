# 焦點、注音鍵盤、媒體 UI 與正版動畫來源設計

## 目標

修正設定頁與虛擬鍵盤不符合直覺的方向操作；讓動畫、YouTube 與 Bilibili 擁有清楚一致、適合遙控器的 tvOS 內容介面；新增動畫瘋及官方 YouTube 動畫頻道等正版入口。動畫瘋播放必須使用官方播放器、保留廣告與登入限制，同時支援遙控器。

## 不變條件

- 所有修改直接在既有 SwiftUI/AppState/runtime controller 架構內完成。
- 不解析動畫瘋 m3u8、token、unlock、AES key 或影片分段。
- 不跳過、隱藏、加速或偽造動畫瘋廣告流程。
- 不複製 `aniGamerPlus` 的 GPL-3.0 原始碼；只將其 SN／集數概念當作資料建模參考。
- CSS1、BT、Dandanplay、Bilibili 與 YouTube 現有播放功能維持可用。
- 方向鍵可到達所有可操作元件，焦點進入捲動區時對應項目必須自動進入畫面。

## 設定焦點

### 根因

設定畫面的實際順序是：介面縮放、壁紙、網頁放大、影片位置、彈幕大小、彈幕速度、彈幕透明度、彈幕密度、憑證與服務。`SettingsFocus.next/previous` 目前把影片位置放在彈幕之後，並在頂端／底端循環，因此按上下時會跳過眼前的項目或跨越整張列表。

### 行為

- 焦點順序必須與畫面由上到下完全一致。
- 在第一項按上保持第一項；在最後一項按下保持最後一項，不循環。
- 左右只調整可調值；命令型項目仍以 OK 執行，右鍵不得意外開啟檔案面板或重新載入憑證。
- `ScrollViewReader` 只在焦點離開可視範圍時捲動，使用 `.center` 保持上下內容脈絡。

## 標準大千注音鍵盤

### 排列

注音頁改成台灣標準大千鍵盤的四列配置：

1. `ㄅ ㄉ ˇ ˋ ㄓ ˊ ˙ ㄚ ㄞ ㄢ ㄦ`
2. `ㄆ ㄊ ㄍ ㄐ ㄔ ㄗ ㄧ ㄛ ㄟ ㄣ`
3. `ㄇ ㄋ ㄎ ㄑ ㄕ ㄘ ㄨ ㄜ ㄠ ㄤ`
4. `ㄈ ㄌ ㄏ ㄒ ㄖ ㄙ ㄩ ㄝ ㄡ ㄥ`

第五列放空格、刪除、搜尋、ABC 與取消。拉丁鍵盤維持 QWERTY。

### 空間焦點

上下移動不能只沿用相同 array index。鍵盤為每個 key 定義視覺寬度與中心位置；上下時選擇相鄰列中水平中心最接近的 key。這使字元列進入較寬的操作列時，焦點會落到視覺上最接近的按鈕。

組字完成後按上進入候選列；候選列左右移動、按下回到原鍵盤位置。切換 ABC／注音時焦點回到新 layout 的第一個字元，避免停在不相干的操作鍵。

## 共用媒體介面

新增共用 tvOS 媒體元件，而不是讓三個 runtime 各自畫不同的標題與卡片：

- `TVOSMediaTopNavigation`：畫面上方置中的黑色圓角分類列；聚焦／選中分類使用近白底深字。
- `TVOSMediaHero`：以焦點內容的橫幅或封面建立大圖背景、深色漸層、標題與主要操作。
- `TVOSMediaShelf`：橫向內容列，焦點卡片上浮並自動水平捲動。
- `TVOSVideoGrid`：四欄 16:9 縮圖牆，縮圖下方顯示標題、作者／頻道與輔助資訊。
- `TVOSMediaEmptyState`：登入不足、無資料與 API 失敗時使用一致的狀態畫面。

所有共用元件沿用 tvOS 18 深色、霧面、白色焦點樣式，不加入 Liquid Glass。

## 動畫 UI

- 頂部分類：推薦、搜尋、收藏、來源。
- 推薦首頁先顯示焦點作品 Hero，再顯示「繼續觀看」、「最近更新」、「熱門作品」等橫向內容列。
- 作品卡以封面為主，焦點時 Hero 同步更新；不再把整頁做成密集小海報牆。
- 詳情頁使用大幅背景、作品資訊和醒目的「選集」操作；選集以橫向長方形列顯示。
- 收藏第一階段使用現有觀看紀錄衍生內容；若沒有收藏資料，顯示清楚的空狀態，不偽造收藏。
- 來源頁導向既有動漫來源管理，不在瀏覽頁複製另一套來源設定。

## YouTube UI

- 頂部分類：推薦、熱門、訂閱、搜尋。
- 使用四欄 16:9 縮圖牆；每張卡顯示影片標題、頻道名稱與必要的時長／日期資訊。
- 焦點縮圖提亮、上浮並更新上方 Hero 或背景；列表移動自動捲動。
- 未配置 YouTube API 時保留既有 fallback，並在原生空狀態中說明，不退回雜亂網頁。
- 播放頁繼續使用既有官方 YouTube embed 與 tvOS 18 HUD。

## Bilibili UI

介面以使用者提供的六張參考圖為基準。

### 首頁與分類

- 頂部分類列：推薦、熱門、番劇、排行榜、動態、我的、搜尋。
- 可用資料以四欄 16:9 影片牆顯示；卡片包含縮圖、時長、觀看／彈幕數、標題、UP 主與日期。
- 番劇內容可使用直式封面；一般影片使用橫式縮圖。
- 分類切換後焦點進入第一張卡，內容捲回頂部。

### 動態與我的

- 已登入且 API 回傳資料時顯示動態與個人資訊。
- 缺少 Cookie、權限或 API 不可用時顯示「需要登入 Bilibili」的原生狀態與設定入口，不製造假資料。
- 我的頁可呈現現有觀看紀錄、稍後再看／收藏入口；只有官方 API 確認可取得的欄位才顯示數字。

### 訂閱與詳情

- 番劇追番使用圖文混排的三欄卡片，包含封面、簡介、地區、觀看進度與集數。
- 一般影片詳情使用左側標題、UP 主、統計和操作，右側大縮圖，下方為標籤與推薦影片列。
- 既有番劇選集、一般影片播放、Cookie 錯誤、地區／會員限制處理維持不變。

### 播放

- 使用全幅影片與底部資訊層；HUD 顯示標題、時間、播放狀態、字幕、彈幕與畫質入口。
- 彈幕維持由右往左的固定行移動。

## 正版動畫來源

### 來源類型

新增「官方 Web 播放」來源類型，與可解析為 AVPlayer URL 的 CSS1/BT 分開。來源結果包含官方作品 URL、官方集數 URL、服務名稱與地區／登入提示，不包含影片直連。

預設來源：

- 巴哈姆特動畫瘋：原生搜尋、作品與選集 UI；播放使用動畫瘋官方頁面。
- Muse 木棉花官方 YouTube：透過現有 YouTube Data API 搜尋並使用 YouTube 官方 embed 播放。
- Ani-One Asia 官方 YouTube：同上；依影片本身的地區限制顯示結果。

只有能確認為官方發布者的 YouTube 結果才標示為正版來源；一般 YouTube 搜尋結果不自動視為官方動畫。

### 動畫瘋資料

- 獨立實作必要的作品／集數 metadata 解析，不複製 `aniGamerPlus` 程式碼。
- 只取得建立原生作品卡、詳情、選集與官方播放 URL 所需資料。
- 使用節流、timeout、錯誤原因與快取，避免頻繁請求官方網站。
- 網頁結構改變時顯示「動畫瘋頁面格式已變更」，並提供開啟官方首頁的 fallback。

## 動畫瘋官方播放器與遙控器

### 播放容器

`OfficialWebPlaybackView` 使用持久化 `WKWebsiteDataStore` 載入動畫瘋官方集數頁，保留 Cookie、官方登入、廣告、VIP、地區限制和觀看紀錄。原生畫面只保留 Back/Home 狀態與載入／錯誤提示；影片內容和廣告由官方播放器呈現。

第一次按 OK 時嘗試觸發網站自己的播放／全螢幕控制。如果 WebKit 禁止自動全螢幕，保持官方播放器可見並提示再次按 OK。

### 遙控器橋接

遙控器只轉成官方頁面可處理的標準鍵盤事件，不直接操作 `HTMLMediaElement.currentTime`：

- OK、Play/Pause → `Space`／`K`
- 左、右 → `ArrowLeft`／`ArrowRight`
- 上、下 → `ArrowUp`／`ArrowDown`
- Menu/Back → 先送出 Escape 退出全螢幕；第二次返回原生選集
- Home → 直接返回主畫面

按鍵事件送給目前 active element、document 與 window，讓官方播放器決定是否播放、暫停、調音量或快轉。廣告期間是否接受任何操作完全由官方播放器控制；本 App 不偵測廣告倒數、不改時間、不點擊跳過廣告按鈕。

### 失敗處理

- 登入失效：保留官方登入畫面並顯示原生提示。
- 地區／年齡／VIP 限制：顯示官方頁面與簡短原生狀態，不嘗試繞過。
- 官方播放器 DOM 或快捷鍵變更：退回可由虛擬游標操作的完整官方頁面。
- WebKit 無法播放：提供「用預設瀏覽器開啟」而非解析串流。

## 測試

- SettingsFocus 測試必須驗證完整視覺順序與上下邊界不循環。
- 注音 rows 必須精確符合四列大千排列；不同長度列之間上下移動選擇最近中心。
- 動畫、YouTube、Bilibili source contracts 驗證共用 top navigation、Hero、shelf/grid 與 stable scroll IDs。
- Bilibili 驗證分類切換、四欄 grid、登入空狀態、詳情與播放狀態不破壞既有 API 測試。
- 正版來源驗證只產生官方頁面／YouTube embed，不出現 m3u8、token、unlock、AES 或廣告跳過流程。
- OfficialWebPlaybackView 測試所有遙控器鍵映射，並驗證程式碼不設定 `currentTime`、不尋找 skip-ad 元件。
- 完整執行 `swift run TVShellChecks`、`swift build --target TVShellCore` 與 `git diff --check`。
