# CSS1 選集驗證回歸修復設計

## 根因

多來源播放線合併讓更多 CSS1 網站的選集結果進入畫面。部分網站的 selector 會涵蓋作品資訊卡，現有 `isEpisodeAnchor` 只要看見「第 N 集」或純數字便接受，因此「第13集 豆瓣評分」、「更新至第25集」和年份「2021」被誤判成選集。

## 修正

- 保留 CSS1 多來源播放線與高畫質排序。
- 選集標題含豆瓣、評分、更新至、上映、年份等作品資訊詞時拒絕。
- 1900–2099 的四位純數字視為年份而拒絕。
- 純數字選集只在真正的選集區塊或 URL 具有 `play`、`episode`、`watch` 播放路徑時接受。
- 「第 N 話／集」同樣必須位於選集區塊或播放路徑；不再把 `detail` 類作品頁當播放頁。

## 驗證

測試詳情頁同時包含正常第 1 話、豆瓣評分卡、更新至第25集與 2021 年份卡，解析結果必須只保留第 1 話。既有 CSS1 高畫質、並行搜尋、Dandanplay 與完整 `TVShellChecks` 皆須通過。
